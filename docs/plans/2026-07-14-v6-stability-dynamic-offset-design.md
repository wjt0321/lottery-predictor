# V6 稳定性验证与动态科学偏移设计

日期：2026-07-14

## 1. 目标

在保持 `team` 模式固定 5 注、娱乐用途和既有归档 KV 格式不变的前提下，把科学偏移从“每期固定替换 2 球”升级为“有证据才偏移”的动态策略，并建立可以跨窗口、跨随机种子验证其稳定性的完整评估链路。

本轮不增加新专家、不恢复 LSTM、不根据单次回测结果反复调参。

## 2. 动态 0/1/2 偏移策略

默认 `anti_ticket_strategy` 从 `scientific` 升级为 `dynamic`，同时保留：

- `legacy`：旧反共识随机混合基准；
- `scientific`：固定使用 `anti_ticket_red_count` 个科学偏移球的兼容模式；
- `dynamic`：分别求解 1 球与 2 球约束组合，根据预先配置的置信门槛选择 0/1/2。

动态判定使用以下事前信息：

- 最佳组合的科学评分；
- 最佳与次佳组合分差；
- 选中候选覆盖的独立专家数量；
- 是否需要放宽和值约束；
- 结构质量和模型分歧。

规则顺序：

1. 两球方案达到高门槛、证据覆盖足够且未放宽和值时，使用 2 球偏移；
2. 否则一球方案达到基础门槛时，使用 1 球偏移；
3. 否则保留原最低信念矩阵票，即 0 球偏移；
4. 显式 `scientific` 无可行组合时仍回退 `legacy`，确保兼容；动态模式无证据时不随机偏移。

无论选择 0/1/2，第 5 注仍固定为最低信念矩阵行对应的角色票。

## 3. 约束组合选择器

将现有只支持 2 球的选择器泛化为 1 或 2 球：

- 保留基础票中分数最高的 `6-k` 个核心球；
- 从科学候选画像中枚举 `k` 球组合；
- 继续执行合法性、奇偶、三区、和值分位、与前 4 注最大重叠约束；
- 输出最佳分、次佳分、分差、证据专家覆盖和约束放宽状态；
- 相同输入和 seed 下完全可复现。

## 4. 反事实归因

第 5 注解释中保存未替换的原矩阵票：红球、蓝球、矩阵行 ID。回测据此计算：

- 动态偏移 0/1/2 启用次数；
- 科学偏移票平均分与原矩阵票平均分；
- 平均分增量；
- 实际 best-of-5 与“换回原矩阵票”的 best-of-5 差值；
- 改善、变差、持平样本数；
- 红球命中数增量。

这样可以区分“整个五注组合表现好”与“偏移票本身带来增量”。

## 5. 多窗口、多种子稳定性回测

新增只读 CLI：

```bash
python predict.py --team-stability-backtest \
  --stability-windows 36,72,144,288 \
  --stability-seeds 7,42,2026,20260714
```

同一窗口与 seed 下并排运行 `dynamic` 和 `legacy`，默认使用 clean runtime，不加载当前归档补丁。报告包括：

- 每个 run 的完整 team 指标；
- 综合目标分；
- dynamic - legacy 配对差值；
- 各指标均值、标准差、最差值和最好值；
- dynamic 的正向 run 比例；
- 稳健分 `mean_objective - 0.5 * std_objective`；
- 预先固定的验收护栏，不自动调参。

综合目标用于比较而非宣称概率预测能力：红 2+ 15%、红 3+ 30%、红 4+ 30%、最终蓝球 10%、best-of-5 归一化分 15%，并对过高重叠施加小额惩罚。

## 6. 蓝球排名校准诊断

每张最终票保存统一的 `blue_score`。回测按每期最终不重复蓝球的分数排名，累计：

- rank 1～5 的曝光数、命中数和命中率；
- top-1 / top-3 命中率；
- 命中时平均排名；
- 高排名蓝球是否比低排名蓝球表现更好。

这是排序校准诊断，不把引擎原始分数错误解释成真实概率。

## 7. 验收条件

- `team` 始终输出 5 注合法 `6+1`；
- dynamic 的第 5 注只能为 `dynamic_offset_0/1/2`；
- 0 偏移时号码与原矩阵票一致；
- 1/2 偏移时保留 5/4 个最高置信核心球；
- 反事实票信息可 JSON 序列化并进入既有解释归档；
- stability 回测不写归档、相同输入可复现；
- 原有 `legacy`、固定 `scientific`、`team-cover` 和补丁回灌行为继续可用；
- 全量测试通过；
- 代表性稳定性实验完整报告，不根据结果做事后搜索。
