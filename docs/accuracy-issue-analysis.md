### 2.2 蓝球根因诊断

1. **core_blue_pool_size = 3**：核心蓝球池只保留 3 个号，覆盖太窄
2. **蓝球池选择逻辑集中在前几号**：当前 `blue_pattern` 分析倾向于重复热号（01/02/05/07/08/10/16），冷号权重极低
3. **没有独立蓝球引擎**：蓝球预测完全依附于红球策略 Agent，缺乏针对性分析
4. **冷号追号机制过弱**：`missing > 20` 期仅给 `1.15` 倍加权，被热度权重（最高可达 5x+）完全淹没
5. **遗漏值未充分利用**：`analyze_blue_missing()` 的结果在 team 模式中未被有效利用

## 三、红球问题分析

### 3.1 红球频率分布

全历史（317 期）红球频率：
- 高频 (>65)：02(77), 13(69), 09(68), 17(67), 03(66), 06(65), 18(65), 22(65), 30(63)
- 低频 (<50)：21(44), 28(47), 31(47), 29(48), 20(49), 26(49), 12(50)

### 3.2 红球根因诊断

1. **core_red_pool_size = 10 太窄**：只覆盖 30% 的红球候选空间，遗漏大量有效号码
2. **zone Agent 一家独大**：权重稳定度极低，导致所有预测高度同质化
3. **Agent 策略同质化**：8 个 Agent 都围绕热度/遗漏/区间三个维度，缺乏维度的真正多样性
4. **旋转矩阵 10_red_guard_6_to_5 局限性**：10 球池覆盖 6 保 5，但 10 球本身覆盖不足

### 3.3 近期红球模式（最近20期）

- 平均和值：102.7 ± 24.5
- 常见奇偶比：3:3 (40%), 2:4 (20%), 4:2 (15%)
- 常见大小比：3:3 (35%), 4:2 (30%)
- 常见区间分布：2-2-2 (30%), 1-2-3 (20%)
- AC 值范围：4-10，常见 6-8
- 连号出现率：约 70% 的期次有至少一组连号

## 四、自学习闭环问题

### 4.1 当前状态

- `param_patch` 永远走 default → 参数从未被数据驱动调整
- `weight_patch` 的调整幅度过小（step=0.02），无法对抗 zone Agent 的压倒性优势
- 归档分析器 `analyze_archive.py` 的排行榜没有反馈到实际预测链路中
- `hot_cold_window` 固定 40 期，不适应市场变化

### 4.2 修复方向

1. 每期开奖后自动更新各 Agent 的权重
2. hot/cold 窗口自适应调整
3. 引入贝叶斯概率更新：每期开奖后更新蓝球概率分布

## 五、改进方案总结

| 优先级 | 改进项 | 预期效果 |
|--------|--------|----------|
| P0 | 蓝球独立预测引擎 | 提高蓝球覆盖面，纳入冷号追号 |
| P0 | 蓝球候选池扩大到 5-8 个 | 增加蓝球多样性 |
| P1 | 红球候选池扩大到 14 个 | 提高红球覆盖命中率 |
| P1 | Agent 投票去同质化 | 减少 zone 一家独大 |
| P1 | 旋转矩阵适配 14 球池 | 适配更大的候选池 |
| P2 | 自学习闭环参数自适应 | 数据驱动参数调整 |
| P2 | hot/cold 窗口自适应 | 适应市场变化 |
